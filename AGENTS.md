## Project Map (MAP.md)

This repo maintains a feature map in `MAP.md`. Rules for AI assistants:

1. **Read first.** At the start of every session, read `MAP.md` before
   exploring code. Use it to locate features instead of scanning the repo.
2. **Update in the same commit.** If you add, change, or remove a feature,
   update its section in `MAP.md` within the same commit. This is part of
   the definition of done, not a separate task.
3. **New feature = new section.** Use the template: heading `### Name {#kebab-id}`
   under the right `## Layer:` layer, then `**What:**` (1-2 sentences),
   `**Files:**` (entry-point paths), `**Depends:**` (anchor links or —),
   `**Status:**` (active | wip | deprecated). Max 6 lines per feature.
4. **What and where, never how.** Implementation details live in code, not in the map.
5. **Keep 3 levels.** Project → Layer → Feature. Never add deeper headings.
6. **Validate.** Run `featmap check` after editing MAP.md if available.
7. **Don't edit** `**Used by:**` lines — they are generated by `featmap links`.
